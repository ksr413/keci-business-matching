# 2025 공공기관 내수구매상담회 매칭 로직

## 개요

구매담당자(바이어)와 참가기업 간의 상담 매칭을 자동화하는 알고리즘
- 바이어: 83명 (2개 날짜로 나뉨)
- 기업: 71개사
- 라운드: 4개 (1, 2, 3, 4)

---

## 데이터 구조

### 구매담당자 리스트 (`2025 구매담당자 리스트.xlsx`)
**시트**: `구매담당자 명단`

| 컬럼 | 필드명 | 설명 |
|------|--------|------|
| A | 번호 | 바이어 ID (A1~A83) |
| B | 소속 | 소속 기관/부처 |
| C | 부서 | 부서명 |
| D | 성명 | 바이어 성명 |
| E | 관심품목 | 구매 관심 품목 (세부) |
| F | 관심 기업 | 관심 기업 번호 (쉼표 구분) |

**예시**:
```
A5 (민준규) → 관심 기업: 1,22,14,7,15,2
A10 (이정아) → 관심 기업: 28,34,61
```

### 참가기업 리스트 (`2025 참가기업 리스트_수정.xlsx`)
**시트**: `Sheet1`

| 컬럼 | 필드명 | 설명 |
|------|--------|------|
| A | 연번 | 기업 ID (1~71, 결번 있음) |
| B | 회사명 | 기업명 |
| C | 주요품목 | 판매 상품/서비스 (세부) |
| D | 품목분류 | 대분류 (전기, 기타 등) |
| E | 관심 구매담당자 | 관심 바이어 번호 (쉼표 구분) |

**예시**:
```
연번 1 (Tidy) → 관심 구매담당자: A66
연번 8 (자재수급원) → 관심 구매담당자: A1, A34, A40, A58, A76
```

## 매칭 우선순위 (5단계)

### 1️⃣ Pool1: 상호 관심 (최우선)
**조건**: 바이어 관심 기업번호 **AND** 기업의 관심 구매담당자 일치

```
바이어 A5의 관심 기업: 1, 22, 14, 7, 15, 2
기업 22의 관심 구매담당자: A5, A51, ...
→ (A5, 기업22) = 상호 관심 → Pool1
```

**우선순위**: 최고
**색상**: 🟢 초록 (상호희망)

---

### 2️⃣ Pool2: 바이어 우선
**조건**: 바이어 관심 기업번호 (기업이 바이어를 선택하지 않아도 됨)

```
바이어 A3의 관심 기업: 8, 12, 4
기업 8이 A3을 선택하지 않았어도 Pool2에 포함
→ (A3, 기업8) = 바이어 우선 → Pool2
```

**우선순위**: 높음
**색상**: 🔵 파랑 (바이어전용)

---

### 3️⃣ Pool3: 기업 우선
**조건**: 기업의 관심 구매담당자 (바이어가 기업을 선택하지 않아도 됨)

```
기업 1의 관심 구매담당자: A66
바이어 A66이 기업 1을 선택하지 않았어도 Pool3에 포함
→ (A66, 기업1) = 기업 우선 → Pool3
```

**우선순위**: 중간
**색상**: 🟠 주황 (기업전용)

---

### 4️⃣ Pool4: 세부품목 일치
**조건**: 바이어 관심품목 **AND** 기업 주요품목 일치

```
바이어 관심품목: "이동통신 네트워크"
기업 주요품목: "이동통신 네트워크 관리 시스템"
"이동통신 네트워크" 키워드 매칭 → Pool4
```

**우선순위**: 낮음
**색상**: 회색 (품목매칭)

---

### 5️⃣ Pool5: 대분류 일치
**조건**: 바이어 관심품목 **AND** 기업 품목분류(대분류) 일치

```
바이어 관심품목: "전기"
기업 품목분류: "전기"
대분류만 일치 → Pool5
```

**우선순위**: 최저
**색상**: 회색 (품목매칭)

---

## 매칭 알고리즘

### 단계 1: 풀 분류
모든 (바이어, 기업) 쌍을 Pool1~5 중 하나로 분류

### 단계 2: 그리디 라운드 배정
```
FOR 각 풀 (순서: Pool1 → Pool2 → Pool3 → Pool4 → Pool5):
    WHILE 변화가 있음:
        FOR 각 라운드 (1, 2, 3, 4):
            FOR 각 (바이어, 기업) 쌍:
                제약 조건 검증:
                [C1] 바이어 슬롯 남아있는가? (최대 4)
                [C2] 이 쌍이 이미 배정되었는가?
                [C3] 이 바이어가 이 라운드에 이미 배정되었는가?
                [C4] 이 기업이 이 라운드에 이미 배정되었는가?

                모든 제약 충족 → 배정
```

### 제약 조건

| 제약 | 설명 | 예시 |
|------|------|------|
| [C1] | 바이어 슬롯 제한 | 바이어당 최대 4회 매칭 |
| [C2] | 쌍 중복 금지 | (A1, 기업5)는 1번만 매칭 |
| [C3] | 라운드 내 바이어 중복 금지 | 라운드1에서 A1은 1번만 배정 |
| [C4] | 라운드 내 기업 중복 금지 | 라운드1에서 기업5는 1번만 배정 |

## 출력 결과

### 시트 구조 (6개)
1. **기관별 라운드 배정 (매칭유형)** - 색상으로 매칭유형 표시
2. **기관별 라운드 배정** - 기본형
3. **라운드별 매칭** - 상세 데이터
4. **통계** - 전체 통계
5. **미배정** - 배정되지 않은 쌍
6. **경고** - 데이터 이상 사항

### 기관별 라운드 배정 컬럼
```
참가날짜 | 소속 기관 | 부서 | 성명 | 관심품목 | 라운드1(색상) | 라운드2 | 라운드3 | 라운드4
```

### 색상 코드
- 🟢 **#C6EFCE** (초록): Pool1 상호희망
- 🔵 **#BDD7EE** (파랑): Pool2 바이어우선
- 🟠 **#FCE4D6** (주황): Pool3 기업우선
- 회색: Pool4, Pool5 (품목매칭)

## 검증 기준

### 하드 제약 확인
- ✅ 동일 (바이어, 기업) 쌍 중복 없음 [C2]
- ✅ 동일 바이어가 같은 라운드에 2회 이상 배정 안 됨 [C3]
- ✅ 동일 기업이 같은 라운드에 2회 이상 배정 안 됨 [C4]
- ✅ 바이어당 최대 4회 배정 [C1]

### 풀 처리 순서 확인
- ✅ Pool1 → Pool2 → Pool3 → Pool4 → Pool5 순서로 처리
- ✅ 각 풀 내에서 라운드 1→4 순서로 처리

### 날짜 검증
- ✅ 같은 날짜의 바이어-기업만 매칭
- ✅ 2026-05-20: 41명 바이어
- ✅ 2026-05-21: 42명 바이어
